Lexical Scoping =

In JS, only functions create a new scope (Not just any curly braces).
Ex: function my_function (){ var x // local }. OR Even arguments in parameter is local

This means 2 things:

1) everytime the function is called - new copy of var x is created. This is isolated from other function call and dissipate when function is finished

2) This only applies to "var" keyword. If you instead define "let", this is local to the enclosing block, but not function scope like "var"

Closure

To understand Closure you need to think of 2 properties of functions we saw above:

1) Function can be treated as values (passed as argument, returned and assigned)

2) Local variables are recreated everytime function is called

This means that you could return a function that captures (freezes) the copy of the local variable from explicit declaration of local variable or parameter. The returned function can be used to perform rest of the computations. This is CLOSURE [ function enclosing local variable ]

Functions and Side effects [pure vs impure]

Pure functions are = given same argument it will return same output. Produces no side effects

Impure functions are = output does not depend on input. given same input it produces different output. Eg: Math.random();

Pure functions have many beneficial properties
1) completelyn independent of outside state (immune to entire classes of bugs that has to do with shareed mutable states)

2) Independent nature makes them great candidate for parallel processing (essential for many resource-intensive computing tasks)

3) Make code adaptable for future changes

Mutability

Numbers, Strings and Booleans are immutable (impossible to change even with reference). Object, on the other hand, its value of the content can be modified by changing its properties.
